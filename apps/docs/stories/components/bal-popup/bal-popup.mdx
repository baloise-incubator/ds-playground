## Usage

To link the popup component to a trigger element, you need to assign an ID to your popup component and add the bal-popup attribute to the trigger element, as shown in the example below:

```html
<bal-button bal-popup="my-popup">Click me</bal-button>
<bal-popup id="my-popup" label="Popup Label">Popup content</bal-popup>
```

Furthermore, most of the properties of the popup component can be set on the trigger element. Here's an HTML snippet demonstrating the usage:

```html
<bal-button bal-popup="my-popup" bal-popup-position="top" bal-popup-arrow>Click me</bal-button>
<bal-popup id="my-popup" label="Popup Label">Popup content</bal-popup>
```

In this example, the bal-button element serves as the trigger element for the popup component.
It includes the bal-popup attribute, which references the ID of the popup component, "my-popup". Additionally, it specifies the position of the popup using bal-popup-position="top" and enables the arrow indicator with bal-popup-arrow.

The actual content of the popup is defined within the bal-popup element.
It includes a label attribute, "Popup Label", for accessibility purposes and to show a heading.

## Variants

The popup component offers three variants: Popover, Fullscreen, and Drawer.

- The **Popover** variant, which is the default, functions similarly to a tooltip but allows for displaying more extensive content.
- The **Fullscreen** variant behaves like a modal, covering the entire window and is particularly useful for mobile solutions.
- Lastly, the **Drawer** variant slides up from the bottom of the window.
